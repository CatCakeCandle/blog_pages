进度管理

管理过程

规划进度管理
Plan Schedule Managerment

为规划,编制,管理,执行和控制项目进度而制定的政策,程序和文档的过程

输入

项目管理计划
项目章程
事业环境因素因素和组织过程资产

工具技术

专家判断
分析技术
会议

输出

进度管理计划

定义活动
Define Activities

识别和记录为完成项目可交付成果而采取的具体行动的过程

输入

进度管理计划
范围基准
事业环境因素 和组织过程资产

工具技术

分解
滚动式规划
专家判断

输出

活动清单,活动属性
里程碑清单 (项目中重要的时间点或事件)

排列活动顺序
Sequence Activeties

定义:识别和记录项目活动之间逻辑关系的过程<br>作用: 定义工作之间的逻辑顺序,以便在机顶的所有项目制约因素下获得最高的效率

估算活动资源
Estimate Activity Resources

定义: 估算活动资源就是确定在实施项目活动时要使用何种资源(人员.设备或物资),每一种实用的数量,以及何时用于项目计划活动<br>作用: 明确完成活动所需要资源种类,数量和特性, 以便于做出更准确的城北和持续时间估算.<br>注: 估算互动资源过程要同成本估算过程紧密配合

项目日历

编排开展激活活动的工作日或轮流班次, 以及不开展计划活动的非工作日的日历. 项目日历影响到所有活动 (简单理解就是 项目的工作日与非工作日)

资源日历

表明没中具体资源的可用工作日或者工作班次的日历. 资源丽日规定了在项目期间特定资源何时使用,可用多久. 可以在活动或者项目层面建立资源日历.<br>另外还需要考虑更多的资源属性, 例如经验和/或技能水平.

工具技术

专家判断(Export Judgment)
备选方案分析(Alternative Analysis)
发布的估算数据(Published estimates)
自下而上估算(Bottom-Up Estimating)
对工作组成部分进行估算的一种方法. 准确性取决于打个活动或工作包的规模和复杂程度
项目管理软件

估算活动持续时间
Estimate Activity Durations

定义:根据资源估算的结果, 估算完成单项活动所需所需要工作时段的过程<br>作用: 确定每个活动所需要花费的时间量, 未指定进度计划过程提供主要输入

工具技术

专家判断(Export Judgment)
类比估算(Analogous Estimating)
以过去类似项目的参数值(如持续时间,预算)
参数估算(Parametric Estimating)
利用历史数据和其他变量(如代码行)之间统计关系.来估算诸如成本,预算和陈继续时间等活动参数.
三点估算(Three-Point Estimate)
通过考虑估算中的不确定性和风险, 可以提高活动持续时间估算的准确性,表明持续时间估算的变化范围<br>估算值 = (最乐观时间 + 4 X 最可能时间 + 最悲观时间 ) /6
储备分析(Reserve Analysis)
(储备量用来应对风险)在进行持续时间估算时,需考虑应急储备(时间储备,缓冲时间) 并将其纳入项目进度计划中,用来应对进度方面的不确定性.

制定进度计划
Develop Schedule

概念

分析活动顺序,持续时间,资源需求和进度制约因素,创建项目进度模型的过程
作用: 把活动,持续时间,资源,资源可用性和逻辑关系带入进度规划工具,从而形成包含各个项目活动的计划日期的进度模型
时间约束因素
活动开始或结束的强制性日期.
项目发起人客户或其他干系人制定关键时间或里程碑
辅助数据和信息
里程碑
计划活动
活动属性
假设与约束因素
按时间段提出的资源需求
其他可供选择的进度计划
进度应急储备

步骤

1. 项目描述<br>2. 项目分解与活动界定<br>3. 工作描述<br>4. 项目组织和工作责任分配<br>5. 工作排序<br>6. 计算工作量<br>7. 估算工作持续时间<br>8. 绘制网络图<br>9. 进度安排

输入

进度管理计划, 项目范围说明书,风险登记册
活动清单,活动属性,项目进度网络图,活动资源需求, 资源日历
互动持续时间估算,项目人员分派,资源分解结构
事业环境因素 ,组织过程资产

工具技术

关键路径法
关键链法
CCM 统一管理缓冲
计划评审技术(PERT)
6希格玛
进度压缩
资源优化技术 (资源平衡,资源平滑)
资源平衡(导致关键路径的改变) (先做)
 Resource Leveling
为了在资源需求与资源供给之间去的平衡, 根据资源制约对开始日期和结束日期进行调整的一种技术, 往往会导致关键路径的改变, 通常是延长
资源平滑(不会改变关键路径) (后做)
 Resource Smoothing
对进度模型中的活动进行调整,从而使项目资源需求不超过预订资源限制的一种技术, 不会改变关键路径,完工日期不会延时, 可能无法实现所有资源的优化.<br>通常是把资源看成可替换的.

输出

进度精准
项目进度计划 (项目进度网络图, 横道图,里程碑图)
进度数据,项目日志,项目管理计划更新,项目文件更新

控制进度
Control Schedule

监督项目活动状态, 更新项目进展, 管理进度基准变更,以实现计划的过程

内容

判断项目进度当前状态
对引起进度变更的因素施加影响,以保证这种变化朝着有利的方向发展.
判断进度是否发生变更
发生变更时候按照变更控制流程管理实际的变更

步骤

1.分析进度,找出那些地方需要采取纠正措施
2. 确定应采取哪些具体纠正措施
3. 修改计划, 将纠正措施列入计划
4. 重新计算季度,估计计划采取的纠正措施的效果.

工具

相交曲线
ES 曲线+ LS 曲线 (最早开始,最晚开始)
比较曲线

进度更新

1. 分析偏差产生的原因 
人员因素
工具,设备因素
方法,技术因素
资金因素
环境因素
2. 分析偏差对进度产生的影响
产生偏差活动是否为关键活动
偏差是否大于总浮动时间
偏差是否大于自由浮动时间
3. 调整项目进度计划
赶工(增加资源)
以最小的成本最大限度的压缩进度
加人,加班
快速跟进(并行)
效率增加,风险增加, 而且不一定能适用
缩减项目范围
分批交付,或者缩减范围, 必须要有客户方的认可
使用高素质的资源或者经验更丰富的人员
改进方法或进度
加强质量管理,减少返工
