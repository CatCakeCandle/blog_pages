项目管理-范围管理

Posted on 2021-05-12 Edited on 2021-05-14

范围管理
Project Scope Management

概念

产品范围

表示产品或者服务的特性和功能, 如何确定信息系统的范围在软件工程中叫做需求分析

项目范围

为了完成具有所规定特征呵呵功能的产品和服务必须完成的全部工作

项目的基准范围

使经过品准的详细的项目范围说明书,WBS 和 WBS 词典

作用

项目范围管理不仅仅使让项目管理和实施人员知道未达到预期目标需要完成那些具体工作,还要确认请相互项目项管各方在没想工作中清晰的分工界面和责任.

管理过程 6个

1. 规划范围管理
Plan Scope Managerment

规划过程组
创建范围管理计划,书面描述如何定义,确认和控制项目范围的过程

输入

项目管理计划
项目章程
事业环境因素
组织过程资产

工具技术

专家判断 Export Judgment
会议

输出

范围管理计划
需求管理计划

2. 收集需求
Collect Requrements

规划过程组
定义:为实现项目目标而确定,记录并管理干系人的需要和需求的过程 作用:为定义和管理项目范围(包括产品范围) 奠定基础

输入

范围管理计划
需求管理计划
干系人管理计划
项目章程
干系人登记册

工具技术

访谈, 焦点小组,引导式研讨会, 问卷调查,观察
群体创新技术(头脑风暴,名义小组,Deiphi,思维导图)
群体决策技术, 原型法,标杆对照, 系统交互图,文件分析

输出

需求文件
需求跟踪矩阵

3. 定义范围
Define Scope

规划过程组
#### 概念 
制定项目和产品详细描述的过程
明确所收集的需求那些将包含在项目范围内, 那些将排除在项目范围外,从而明确产品,服务或成果的边界
项目范围说明书
内容
产品范围描述
验收标准
可交付成果
项目的除外责任(那些不属于项目范围)
制约因素 (如 强制性日期)
假设条件(不需要验证即可视为正确真实的因素)
作用
确定范围
沟通基础(共识)
变更基础(判断是否超界)
规划和控制依据
规划基础(成本,进度等计划的基础)

输入

项目章程
项目范围管理计划
需求文件 &nbsp; (来自于收集需求的输出)
组织过程资产

工具技术

产品分析
备选方案生成
专家判断, 引导式研讨会

输出

项目范围说明书 (范围基线的组成部分)
项目文件更新

4. 创建 WBS 
Create WBS

规划过程组
#### 概念 
作用 : 对所教福的内容提供一个结构化视图
定义: 将项目可交付成果和项目工作分解为较小的,更衣与管理的组件的过程

需要开展的活动

识别和分析可交付成果及相关工作
确定 WBS 结构以及编排方法 (树型,列表型)
自上而下的逐层细化分解
为 WBS 组成部分定制和分配标志编码
核实工作芬姐的程度是恰当的

输入

范围管理计划
项目范围说明书 (前置(定义范围)产出物)
需求文件, 事业环境因素, 组织过程资产

工具技术

分解
专家判断

输出

范围基准
项目文件更新

5. 确认范围
Vaidate Scope

监控过程组
#### 概念 
定义:是正式验收已完成的项目科可交付成果的过程
作用: 使验收过程具有客观性,同事通过验收每个可交付成果,提高最终产品, 服务或成果获得验收的可能性
贯穿于项目整个过程

输入

项目管理计划
需求文件,需求跟踪矩阵
可是的可交付成果, 工作绩效数据

工具技术

检查 (审查, 产品评审,审计, 走查,巡检)
检查工具 关注点
1. 可交付成果是否使可确认的
1. 每个可交付成果是否有明确的里程碑,里程碑是否明确的, 可辩识的事件
3. 是否有明确的质量标准
4. 审核和承诺是否清晰的表达, 并获得一致同意
5. 项目范围是否覆盖需要完成的产品货服务进行的所有活动,有没有遗漏或错误.
6. 项目范围风险是否太高,管理层是否能够降低可预见的更新啊发生时对项目的冲击.
群体决策技术

输出

验收的可交付成果, 工作绩效信息
变更请求,项目文件更新

6. 控制范围
Contorl Scope

监控过程组
定义: 监督项目和产品的范围状态管理范围基准变更的过程
作用: 在振哥项目期间保持对范围基准的维护

涉及内容

影响导致范围变更的因素
判断范围变更是否已经发生
范围变更发生时管理实际的变更
控制范围与其他控制过程相结合

变更原因

项目外部环境发生变化
项目范围的计划编制不够周密详细,有一定的错误或者遗漏
市场上出现了或者设计人员提出了新技术,新手段, 新方案
项目实施组织本身发生变化
客户对项目,项目产品或服务的要求发生变化

输入

项目管理计划
需求文件
需求跟踪矩阵
工作绩效数据
组织过程资产

工具与技术

偏差分析

输出

工作绩效信息
变更请求
项目文件更新,组织过程资产更新
